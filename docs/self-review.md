# Self-Review Checklist

## Criteria

- Critical bugs
- Architectural concerns
- Potential logic bugs
- General unsoundness
- Maintainability concerns

## Severity Levels

| Severity | Description |
|----------|-------------|
| High     | Critical issues that must be fixed before merging |
| Medium   | Notable concerns that should be addressed |
| Low      | Minor issues or suggestions for improvement |

## Process

Rank each finding by severity (high/med/low) and present them before taking any action.
