# Roadmap

This folder tracks implementation plans for prosecheck.

## Structure

- **Active project files** — Each `.md` file in this directory represents an active group of work (e.g., `publishing.md`). Tasks are tracked with checkbox items (`- [ ]` / `- [x]`).
- **`archive/`** — Completed roadmap groups are moved here, prefixed by approximate completion date (e.g., `2026-03-05-initial-sprint.md`). Archived files preserve the full task history for reference.

## How to use

1. Before starting work, read the active roadmap files to understand current priorities.
2. When completing a task, check the box (`- [x]`).
3. When discovering new tasks during implementation, add them to the appropriate file.
4. When all tasks in a file are complete, move it to `archive/` with a date prefix.
5. To start a new workstream, create a new `.md` file in this directory.
